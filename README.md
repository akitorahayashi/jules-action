# jules-action

`jules-action` is a TypeScript GitHub Action that creates a Jules API session from workflow inputs.

## Quick Start

```yaml
- uses: akitorahayashi/jules-action@v1
  env:
    JULES_API_KEY: ${{ secrets.JULES_API_KEY }}
  with:
    prompt: Refactor retry logic in the API client
```

## Prompt From File

```yaml
- name: Build prompt from file
  id: prompt
  shell: bash
  run: |
    {
      echo "value<<EOF"
      cat .github/prompts/refactor.md
      echo "EOF"
    } >> "$GITHUB_OUTPUT"

- uses: akitorahayashi/jules-action@v1
  env:
    JULES_API_KEY: ${{ secrets.JULES_API_KEY }}
  with:
    prompt: ${{ steps.prompt.outputs.value }}
```

## Action Contract

Inputs:

- `prompt` (required)
- `source` (optional, omitted: resolved from `GITHUB_REPOSITORY` as `sources/github/{owner}/{repo}`)
- `starting-branch` (optional, omitted: resolved in order `GITHUB_HEAD_REF` -> `GITHUB_REF_NAME`)
- `title` (optional, omitted: generated by the API)
- `require-plan-approval` (optional, omitted: `true`)
- `automation-mode` (optional, omitted: `AUTO_CREATE_PR`)

Required environment variables:

- `JULES_API_KEY`

Outputs:

- `session-name`
- `session-id`
- `resolved-source`
- `session-title`
- `session-state`
- `session-url`

## Runtime Flow

1. Read and validate action inputs.
2. Resolve source and branch from explicit input or workflow context.
3. Call `sources.list` to verify the source.
4. Call `sessions.create` with the normalized payload.
5. Emit created session outputs.
